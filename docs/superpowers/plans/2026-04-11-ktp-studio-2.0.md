# KTP Studio 2.0 & Quad Deskew Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transform the KTP tagging module into an expert "Studio" environment with a spacious viewport, 4-point perspective correction (Quad Deskew), and robust AI auto-binding to Excel data.

**Architecture:** A centralized "Canvas-First" UI in React with floating, high-contrast toolbars. Perspective transformation and OCR are offloaded to a Python bridge sidecar for production-grade image processing.

**Tech Stack:** React (TypeScript), TailwindCSS, Tauri v2, PaddleOCR (via Python bridge), OpenCV (for homography).
